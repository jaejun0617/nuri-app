# 펫동반 장소 메타 스키마 초안

## 1. 목표

현재 NURI의 펫동반 장소 탐색은 외부 API 후보 검색이 중심이다.

이 문서는 외부 후보를 즉시 대체하는 자체 장소 DB가 아니라, 외부 후보 위에 올라가는 `서비스 메타 레이어`를 먼저 설계하기 위한 초안이다.

즉 이번 스키마의 목적은 아래다.

- 외부 후보에 대한 검증 상태 저장
- 운영 메모와 펫동반 정책 보강
- 사용자 제보/북마크 집계
- 향후 운영 검수 source of truth로 승격 가능한 기반 확보

## 2. 설계 원칙

- 내부 PK와 외부 provider id를 분리한다.
- 외부 원본은 `provider + provider_place_id`로 매핑한다.
- Kakao 결과만으로 확정 상태를 만들지 않는다.
- 상태 모델은 앱과 DB에서 1:1로 연결 가능해야 한다.
- 초기 단계에서는 enum보다 `text + check constraint`를 우선 검토한다.

## 3. 권장 메타 테이블

테이블명 제안:

- `pet_place_service_meta`

권장 컬럼:

- `id uuid primary key default gen_random_uuid()`
- `provider text not null`
- `provider_place_id text not null`
- `verification_status text not null`
- `source_type text not null`
- `pet_policy_text text null`
- `operating_status_label text null`
- `admin_note text null`
- `user_report_count integer not null default 0`
- `bookmarked_count integer not null default 0`
- `created_at timestamptz not null default now()`
- `updated_at timestamptz not null default now()`

권장 제약:

- `unique (provider, provider_place_id)`
- `check (verification_status in ('unknown', 'keyword-inferred', 'user-reported', 'admin-verified', 'rejected'))`
- `check (source_type in ('system-inference', 'user-report', 'admin-review'))`

## 4. 상태 모델 매핑

앱 상태와 DB 값은 아래처럼 1:1로 맞춘다.

- `unknown`
  - 외부 후보만 있고 서비스 검증 없음
- `keyword-inferred`
  - 외부 키워드/속성 기반 추정
- `user-reported`
  - 사용자 제보가 존재
- `admin-verified`
  - 운영 검수 완료
- `rejected`
  - 동반 불가 제보 또는 운영 반려

## 5. 왜 enum보다 text + check constraint인가

현재 단계에서 enum보다 `text + check constraint`가 더 적합하다.

- 상태명이 바뀌거나 추가될 가능성이 아직 있다.
- Supabase/Postgres enum은 변경 시 마이그레이션 부담이 더 크다.
- check constraint는 현재 단계의 진화 비용이 낮다.

enum 전환은 상태 집합이 실제 운영에서 충분히 고정된 뒤에 검토한다.

## 6. 외부 매핑 전략

권장 방식:

- 내부 PK: `id uuid`
- 외부 매핑 키: `provider + provider_place_id`

비권장 방식:

- Kakao place id를 내부 PK로 직접 사용

비권장 이유:

- provider 종속성이 강하다.
- Google Places, TourAPI 병행 시 충돌과 이관 리스크가 있다.
- 내부 운영 이력과 외부 원본 수명이 다르다.

## 7. 다음 확장 테이블 후보

메타 본체와 분리 권장:

- `pet_place_user_reports`
  - 사용자 제보 원본
- `pet_place_bookmarks`
  - 사용자별 북마크
- `pet_place_verification_logs`
  - 운영 검수 이력
- `pet_place_search_logs`
  - 검색 로그

이번 단계에서는 집계값만 `pet_place_service_meta`에 두고, 상세 원본은 별도 테이블로 분리하는 방향이 안전하다.
