# 공용 릴리즈 묶음 운영 기준 통합 완성본

## 1. 문서 목적

이 문서는 `docs/sql/공용-릴리즈-묶음/` 아래에 있는 SQL의 역할과 사용 기준을 정리한 문서다.

여기 있는 SQL은 실제 migration 이력과 다르다. 배포 이력 추적보다는 특정 시점에 수동으로 참고하거나, 새 환경을 빠르게 구성하거나, 운영 중 특정 기능을 추가할 때 쓰는 공용 릴리즈 묶음이다.

## 2. 현재 역할

현재 `docs/sql/공용-릴리즈-묶음/`은 두 축으로 나뉜다.

- `전체-초기-구성`
  - 새 Supabase 프로젝트를 한 번에 세팅할 때 보는 기준
- `기능-추가`
  - 운영 중 필요한 기능을 별도로 붙일 때 참고하는 SQL

즉, 이 폴더는 "실제 배포 이력"이 아니라 "수동 실행용 운영 SQL 라이브러리"에 가깝다.

## 3. 현재 포함된 SQL

### 3.1 전체 초기 구성

- `누리-전체-초기-구성.sql`

의미:

- 기본 enum, 테이블, 함수, RLS, storage bucket/policy를 한 번에 세팅하는 전체 구성본
- 새 프로젝트나 초기 환경 구성에서 참조하기 좋다

### 3.2 기능 추가

- `계정-동의-이력-및-계정-삭제.sql`
- `메모리-카테고리-서브카테고리-가격-추가.sql`

의미:

- 계정 동의 이력 / 계정 삭제 RPC 같은 추가 기능
- `memories.category`, `sub_category`, `price` 같은 별도 컬럼 확장

## 4. 마이그레이션과의 차이

아래 기준으로 구분한다.

- `docs/sql/공용-릴리즈-묶음/`
  - 수동 실행용
  - 공용 재사용 기준
  - 초기 세팅 또는 기능 묶음 참고용
- `docs/sql/마이그레이션/`
  - 실제 배포 이력
  - 날짜 순 누적 관리
  - 운영 적용 순서 추적용

즉, 운영 반영 이력 추적이 중요하면 마이그레이션으로 가고, 특정 상황에서 재사용할 수 있는 공용 SQL이면 여기로 온다.

## 5. 도메인 SQL과의 차이

도메인 SQL은 집사 꿀팁 가이드, 타임라인 메모리 이미지 시스템처럼 특정 작업의 현재 최종 구조를 보여준다.

예시:

- `docs/sql/도메인/집사-꿀팁-가이드/집사-꿀팁-가이드-운영-스키마-최종.sql`
- `docs/sql/도메인/타임라인-메모리-이미지-시스템/타임라인-메모리-이미지-운영-스키마-최종.sql`

반대로 공용 릴리즈 묶음은 특정 도메인 설명보다 더 넓거나, 더 공통적인 SQL을 모은다.

## 6. 추가 원칙

새 SQL이 생기면 아래 기준으로 본다.

1. 배포 이력으로 남겨야 하면 `docs/sql/마이그레이션/`
2. 도메인 현재 최종본이면 `docs/sql/도메인/`
3. 수동 실행용 공용 SQL이면 `docs/sql/공용-릴리즈-묶음/`

공용 릴리즈 묶음에 넣을 때는 아래를 지킨다.

- 폴더명만 보고 목적이 보여야 한다.
- 새 환경 전체 구성인지, 기능 추가인지 먼저 나눈다.
- 도메인 전용 SQL을 여기로 끌어오지 않는다.
- 실제 운영 이력 추적이 필요한 변경은 migration으로 분리한다.

## 7. 현재 운영 관점에서의 의미

- `누리-전체-초기-구성.sql`은 새 프로젝트 세팅을 빠르게 맞출 수 있는 기반이다.
- 기능 추가 SQL은 운영 중 필요한 수동 확장을 빠르게 반영할 때 유용하다.
- 다만 이 폴더만 보고 현재 서비스 전체 상태를 판단하면 안 된다.
- 현재 서비스 구조는 도메인 최종본 문서와 SQL, 그리고 migration 이력을 같이 봐야 정확하다.
