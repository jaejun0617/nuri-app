# 출시 준비도 회복 전체 우선순위 로드맵

상태: 진행 중
최종 갱신: 2026-03-30
판정 기준: 58 / 100

## 배경

- 현재 전체 점수는 58 / 100이다.
- 기능 추가보다 출시 blocker 제거, 운영 증적 확보, 환경 분리, source of truth 정리가 점수 상승에 더 직접적이다.

## 현재 문제

- 정책 문서 외부 공개와 앱 링크가 닫히지 않았다.
- physical-device QA, 계정 삭제, 비밀번호 재설정, 커뮤니티 row-level 증적이 부족하다.
- Guide는 remote와 local seed가 함께 열려 있어 운영 source of truth가 흐린다.
- runtime 설정 중 tracked 파일에 남은 값과 local-only 관리 원칙이 아직 완전히 잠기지 않았다.
- PetTravel은 신뢰도 도메인이라 계약과 증적이 흔들리면 공개 노출 리스크가 크다.

## 목표

- 점수 상승폭이 가장 큰 축부터 순차적으로 잠근다.
- release blocker 트랙과 engineering next slice를 섞지 않는다.

## 작업 목록

1. 액션 1. Release blocker evidence pack
   - 이유: QA/운영 준비도 점수를 가장 크게 끌어올리는 축이다.
2. 액션 2. 정책 문서 외부 공개 및 앱 링크 연결 준비
   - 이유: 법무/정책 미마감 상태는 공개 배포 자체를 막는다.
3. 액션 3. PetTravel trust/search/filter 계약 잠금
   - 이유: 공개 신뢰도 도메인의 계약 흔들림은 직접적인 사용자 오판 리스크를 만든다.
4. 액션 4. Runtime 환경 분리 및 하드코딩 제거
   - 이유: 배포 후 유지보수성과 운영 통제를 위해 반드시 필요하다.
5. 액션 5. Guide source of truth 정리
   - 이유: 운영 콘텐츠 도메인이 test seed fallback과 섞인 상태를 정리해야 한다.
6. 후속. nickname constraint alignment / data audit
   - 이유: 현재 대표 SQL drift이지만 위 1~5보다 출시 준비도 상승 폭은 작다.

## 완료 기준

- 액션 1~5 각각에 대해 상태, blocker, 검증 결과, 다음 액션이 문서로 고정된다.
- release blocker와 engineering slice가 문서상 분리된다.

## 리스크

- 액션 1과 2는 외부 의존성이 있어 이번 턴에 전부 완료될 수 없다.
- 액션 4는 설정 주입 방식을 무리하게 바꾸면 앱 부팅 경로를 깨뜨릴 수 있다.

## 다음 액션

- 액션 1부터 5까지 문서와 실제 변경을 동시에 진행하고, 하위 완료 단위는 완료 기록 문서에 남긴다.
