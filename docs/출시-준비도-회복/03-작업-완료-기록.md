### 완료 ID

- `REL-2026-03-31-PASSWORD-RESET-RECOVERY-SANDBOX-FIX`

### 작업명

- 비밀번호 재설정 recovery session sandbox 가드 및 cleanup guarantee 구현

### 완료일

- 2026-03-31

### 무엇을 했는지

- `authStore`에 persisted `passwordRecoveryFlow` 상태와 TTL을 추가해 recovery 세션을 앱 전역 로그인 상태와 분리했다.
- `AppProviders`에서 recovery session을 일반 로그인 bootstrap으로 승격하지 않게 막고, guest sandbox 상태에서 profile fetch / pet fetch / monitoring user 승격을 차단했다.
- `PasswordResetRecoveryScreen`, `PasswordResetFormScreen`에 recovery 진입/종료 경로를 정리해 성공, 에러, 뒤로가기, 취소 모두 `SignIn` reset으로 수렴시키고 임시 세션을 완전히 정리하게 바꿨다.
- `boot.ts`, `HomeScreen`, `MainScreen`에 2차 방어선을 추가해 recovery active 상태에서 `NicknameSetup`, `PetCreate(from: 'auto')`, 홈 자동 진입을 막았다.
- `SignInScreen`에는 stale recovery flag가 남아 정상 로그인까지 막지 않도록 local recovery cleanup 안전장치를 추가했다.

### 검증 결과
