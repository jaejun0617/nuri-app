# 액션 2. 정책 문서 외부 공개 및 앱 링크 연결 준비

상태: 진행 중
최종 갱신: 2026-03-30
우선순위: P0

## 배경

- 정책 문서 source of truth는 코드에 있지만, 외부 공개 URL과 앱 연결은 아직 닫히지 않았다.

## 현재 문제

- `terms`, `privacy`, `accountDeletion`은 `draft`, `marketing`은 기존에 `pending`이었다.
- 외부 URL은 모두 `null`이다.
- 마케팅 수신 안내는 초안 파일 자체가 없었다.
- 정책 초안 문서 안에 `placeholder` 문구가 남아 있었다.

## 목표

- 앱 연결용 정책 문서 source of truth를 더 완전하게 만든다.
- 외부 공개 전에 필요한 문서/링크/앱 진입점 매핑을 고정한다.

## 작업 목록

- [x] 마케팅 수신 안내 초안 문서 생성
- [x] 정책 초안 문서의 `placeholder` 문구 제거
- [x] `documents.ts`에서 마케팅 문서를 `draft + draftPath`로 정렬
- [x] 정책 source of truth 자동 검증 추가
- [ ] 외부 공개 URL 확정
- [ ] SignUp / More / 삭제 안내에서 실제 외부 링크 열림 검증

## 현재 연결 상태

| 문서 | 코드 상태 | draftPath | 외부 URL | 앱 진입점 |
| --- | --- | --- | --- | --- |
| 이용약관 | draft | `docs/policies/이용약관-초안.md` | 없음 | SignUp |
| 개인정보처리방침 | draft | `docs/policies/개인정보처리방침-초안.md` | 없음 | SignUp |
| 마케팅 수신 안내 | draft | `docs/policies/마케팅-수신-안내-초안.md` | 없음 | SignUp |
| 계정 삭제 안내 | draft | `docs/policies/계정-삭제-안내-초안.md` | 없음 | More Drawer |

## 공개 전 체크포인트

- 외부 URL은 법무/운영 확정 뒤에만 `external`로 바꾼다.
- `external` 전환 시 `url != null`과 앱 링크 실제 열림을 함께 검증한다.
- 필수 문서 2개는 외부 공개 전까지 가입 화면에서 보수적 안내 문구를 유지한다.

## 검증 결과

- `__tests__/legalDocuments.test.ts` 추가 및 통과
- 현재 code source of truth 기준 모든 `draft` 문서는 `draftPath`를 가진다.

## 완료 기준

- 모든 법정/운영 문서가 draft source를 가진다.
- `external` 전환 기준과 앱 진입점이 문서화된다.

## 리스크

- 외부 URL을 임의로 넣으면 허위 완료가 된다.
- 법무 문안 확정 없이 `external`로 바꾸면 운영 리스크가 더 커진다.

## 다음 액션

- 운영/법무가 실제 외부 URL을 주면 `documents.ts` 상태 전환과 앱 열림 검증을 같은 턴에 묶어 진행한다.
