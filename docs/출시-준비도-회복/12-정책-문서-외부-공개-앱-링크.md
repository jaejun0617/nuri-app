## 배경

- 정책 문서 source of truth는 코드에 있지만, 외부 공개 URL과 앱 연결은 아직 닫히지 않았다.

## 현재 문제

- 이전 기준으로는 `terms`, `privacy`, `marketing`, `accountDeletion`이 모두 `draft`였고 앱 링크도 미연결이었다.
- 현재는 앱이 실제로 쓰는 `terms`, `privacy`, `accountDeletion`만 `external`로 전환됐다.
- `marketing`은 이번 범위 밖이라 계속 `draft`다.

## 목표

- 앱 연결용 정책 문서 source of truth를 더 완전하게 만든다.
- 외부 공개 전에 필요한 문서/링크/앱 진입점 매핑을 고정한다.

## 이번 턴 결과

- Notion 웹에 아래 4개 정책 페이지를 실제 생성하고 게시했다.
  - `NURI 이용약관`
  - `NURI 개인정보처리방침`
  - `NURI 계정 삭제 / 탈퇴 안내`
  - `NURI 커뮤니티 운영정책`
- 각 페이지의 공개 URL을 확보했다.
  - 이용약관: `https://amazing-quesadilla-9a8.notion.site/NURI-3364a8f4e2ee8077a3ffc1d5968ddac7`
  - 개인정보처리방침: `https://amazing-quesadilla-9a8.notion.site/NURI-3364a8f4e2ee8045ba30d9ac3c0f3417`
  - 계정 삭제 / 탈퇴 안내: `https://amazing-quesadilla-9a8.notion.site/NURI-3364a8f4e2ee8080abbaf670c1e24ae1`
  - 커뮤니티 운영정책: `https://amazing-quesadilla-9a8.notion.site/NURI-3364a8f4e2ee80218ff1e34e1d1e2a6b`
- 각 URL은 `사이트 보기`로 actual public tab이 열리는지 확인했고, 추가 HTTP 요청 기준 `200` 응답도 확인했다.
- 검증용 상위 페이지 `NURI 정책 문서`도 생성해 4개 링크를 정리했다.
- `src/services/legal/documents.ts`에서 `terms`, `privacy`, `accountDeletion`을 실제 Notion public URL로 연결하고 `status = external`로 전환했다.
- Android 실기기 `SM_S937N`에서 아래 링크 열림을 직접 확인했다.
  - 회원가입 > `이용약관 동의` > `전체 보기` > `NURI 이용약관`
  - 회원가입 > `개인정보처리방침 동의` > `전체 보기` > `NURI 개인정보처리방침`
  - 전체메뉴 > 계정 삭제 > `삭제 안내 상태 확인` > `NURI 계정 삭제 / 탈퇴 안내`
- 세 경로 모두 로그인 요구 없는 public Notion page가 열렸고, Android 뒤로가기 후 앱 복귀도 정상 확인했다.
- 따라서 이번 턴 판정은 `작성 완료 + Notion 반영 완료 + 앱 링크 연결 완료 + 실기기 링크 검증 완료`다.

## 공개 전 체크포인트

- `external` 전환은 `url != null`과 앱 링크 실제 열림이 함께 검증된 문서에만 적용한다.
- 필수 문서 2개는 외부 공개 전까지 가입 화면에서 보수적 안내 문구를 유지한다.
- 운영자 정보, 문의처, 보관기간, 국외 이전 문구는 공개 전 최종 확인이 필요하다.

## 검증 결과

- `__tests__/legalDocuments.test.ts` 기준 `terms`, `privacy`, `accountDeletion`은 canonical Notion URL과 `external` 상태를 검증한다.
- 현재 code source of truth 기준 `draft` 문서는 `marketing`만 남아 있고 `draftPath`를 가진다.
- Notion 게시 URL 4개는 실제로 확보됐다.
- Android 실기기 기준 `terms`, `privacy`, `accountDeletion` 링크 열림과 앱 복귀가 확인됐다.
- `supabase/.temp/project-ref` 기준 linked remote는 `grmekesqoydylqmyvfke`다.
- `supabase migration list --linked` 기준 linked remote와 repo migration은 동기화 상태다.

## 완료 기준

- 모든 법정/운영 문서가 draft source를 가진다.
- `external` 전환 기준과 앱 진입점이 문서화된다.
- Notion 또는 외부 공개 페이지가 실제 생성되고 URL이 기록된다.
- 앱에서 링크 열림이 physical-device로 검증된다.

## 리스크

- `marketing`과 `커뮤니티 운영정책`은 앱 내부 연결 대상이 아니거나 아직 연결되지 않았으므로 같은 완료 상태로 묶으면 안 된다.
- 운영자 정보, 문의처, 법무 확정 문안은 여전히 별도 확인이 필요하다.
- 상위 인덱스 페이지의 public URL은 이번 턴 필수 검증 범위에서 제외했다.
