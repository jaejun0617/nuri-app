# 산책 위치기반 기능 구현 상세

> 최신 신뢰성 개선 기준은 `docs/산책-조회-신뢰성-개선/산책-조회-신뢰성-개선-통합-완성본.md`를 우선 참고한다.
> 실기기 결과 없음 / 키보드 안정성 보강 기준도 위 문서가 최신이다.
> 산책 기능은 현재 `현재 GPS 기반 주변 추천 + 직접 검색`으로 단순화되었고, 다른 동 선택과 고양시 전체 조회는 제거됐다.

## 1. 왜 이 기능이 필요한가

NURI는 반려동물 기록과 케어 중심 기능은 이미 많이 열려 있다. 다음 단계에서는 사용자가 앱 안에서 실제 생활 행동으로 이어질 수 있는 위치 기반 기능이 필요하다.

이번 기능의 목적은 아래 두 가지다.

- 현재 위치 기준으로 바로 쓸 수 있는 `우리동네 산책 / 산책 동선 추천`
- 현재 위치와 직접 검색을 함께 쓰는 `펫동반 카페 / 공간 찾기`

즉, 단순 정보 화면이 아니라 `현재 위치 → 주변 추천 → 검색 → 상세 → 지도/길찾기` 흐름을 실제로 여는 것이 목표다.

## 2. 현재 구조 진단

현재 앱 구조 기준으로 위치 기능은 아래처럼 이미 일부 기반이 있다.

- 공용 위치 훅
  - `src/hooks/useCurrentLocation.ts`
- 좌표 획득 서비스
  - `src/services/location/currentPosition.ts`
- 위치 권한 서비스
  - `src/services/location/permission.ts`
- 행정동 변환
  - `src/services/location/district.ts`
- Kakao REST API 키
  - `src/config/runtime.ts`

즉, 현재 위치와 행정동 변환까지는 이미 구조가 있고, 이번 작업은 그 위에 `장소 탐색 도메인`을 올리는 작업으로 보는 것이 맞다.

전체메뉴 구조도 이미 `MoreDrawer` 기준으로 잘 분리되어 있으므로, 이번 기능은 여기 안에 별도 메뉴 2개를 추가하는 것이 가장 자연스럽다.

## 3. 각각 다른 메뉴로 분리한 이유

이번 기능은 내부적으로 위치/검색/지도 구조를 공유할 수 있지만, 사용자 진입은 분리하는 것이 맞다.

분리 메뉴:

- 우리동네 산책 / 산책 동선 추천
- 펫동반 카페 / 공간 찾기

분리 이유:

- 사용자가 기대하는 목적이 다르다.
- 검색 키워드와 결과 해석이 다르다.
- 빈 상태와 fallback 문구가 다르다.
- 이후 필터와 상세 정보도 도메인별로 달라진다.

즉, 내부 공통 레이어는 공유하되 UX는 분리한다.

## 4. 위치 / GPS 연동 방식

기준 훅:

- `useCurrentLocation`

권한/좌표 흐름:

1. 화면 진입 시 현재 위치 요청
2. 권한 허용 시 좌표 획득
3. 좌표 기준 기본 추천 조회
4. pull-to-refresh 시 재조회
5. 앱 복귀 시 재검증

반드시 구분하는 상태:

- 권한 요청 중
- 권한 거부
- 권한 차단
- 위치 획득 중
- 위치 실패
- 좌표 기반 결과 로딩 중
- 검색 중
- 결과 없음
- 에러 발생

fallback 기준:

- 현재 위치가 없어도 검색은 가능해야 한다.
- 좌표가 없으면 검색어 기반 결과를 먼저 보여준다.
- 마지막 좌표 캐시가 있으면 재사용한다.
- 원본 행정동(`일산3동`)은 유지하되, 위치 기반 탐색 쿼리와 화면 표시에는 숫자동 정도만 정리한 normalize 값(`일산동`)을 사용한다.
- 좌표 캐시가 있으면 먼저 보여주고, 이후 정밀 좌표를 백그라운드로 다시 갱신한다.

## 5. 지도 API 선택 이유

이번 턴 기준 선택:

- 장소 데이터: Kakao Local REST API
- 지도 미리보기: 정적 지도 이미지 provider
- 실제 길찾기/지도 열기: 외부 지도 deep link

이 선택이 맞는 이유:

- 현재 프로젝트에는 Kakao REST 키와 위치 서비스 기반이 이미 있다.
- 장소 검색은 Kakao Local API가 한국 로컬 검색 품질에 맞다.
- 현재 프로젝트에 네이티브 지도 라이브러리가 없고, 이번 턴 목표는 지도 SDK 전체 도입보다 도메인 구조를 안전하게 여는 것이다.
- 따라서 이번 턴은 `provider 분리 + 정적 지도 미리보기 + 외부 지도 열기`가 가장 안전하다.

장기 확장:

- 이후 `react-native-maps` 또는 Kakao/네이버 지도 SDK를 붙이더라도 UI에서 직접 provider를 알지 않도록 서비스 계층을 유지한다.

## 6. 검색 흐름

각 화면 상단에 검색바를 둔다.

- 산책 화면 검색
  - 장소명
  - 공원/호수공원/근린공원 같은 키워드
  - 지역명
- 펫동반 카페/공간 화면 검색
  - 상호명
  - 카페 / 공간 / 지역명
  - 지역명

기본 진입:

- 현재 위치 기반 추천 결과

검색 실행 후:

- 검색어 기준 결과

현재 위치가 없을 때:

- 검색어 기반 탐색만 허용

검색 안정화 기준:

- 한글 조합 입력이 깨지지 않도록 검색창 입력 상태는 즉시 반영하고, 실제 조회만 deferred query로 분리한다.
- 산책 화면은 현재 GPS 기반 주변 추천과 직접 검색에 집중한다.
- 산책 결과는 `가까운순 / 멀리순` 정렬을 직접 바꿀 수 있다.
- 현재 위치 기준 기본 추천은 `현재 좌표 주변 키워드 검색`을 우선 사용해서, 실제 가까운 공원 누락을 줄인다.

추후 확장 가능성:

- 최근 검색어
- 인기 검색어
- 자동완성

## 7. 산책 리스트 / 동선 구조

이번 턴 구현 범위:

- 주변 산책 장소 리스트
- 상세 진입
- 외부 지도 열기
- 상세 하단의 주변 산책 장소 연속 탐색

현재 기본 추천은 `호수공원 / 수변공원 / 생태공원 / 근린공원 / 도시근린공원 / 공원 / 산책로 / 둘레길 / 숲길` 계열만 남기고, `카페 / 음식점 / 반려동물용품 / 출입구 / 화장실 / 주차장 / 학원 / 체험학습장 / 공원시설물 / 생태보존 / 서식지` 같은 잡음 가능성이 큰 결과는 최대한 제외한다. 정렬도 단순 거리순이 아니라 `호수공원 계열 우선 -> 기타 산책 장소 -> 거리순`으로 처리한다.

각 항목 최소 정보:

- 이름
- 거리
- 예상 소요 시간
- 간단 설명
- 좌표

`산책 동선 추천`은 이번 턴에서 실제 네비게이션 경로까지 계산하지 않는다. 대신 아래 수준으로 구조를 연다.

- 시작점 개념
- 추천 포인트/산책장소 개념
- 예상 산책 시간
- 상세에서 지도 열기/길찾기 진입
- 상세 하단에서 현재 리스트 결과 일부를 재사용해 `주변 산책 장소`를 연속 탐색
- 관련 장소 목록은 별도 더보기 없이 바로 이어서 보이되, 과도한 재요청 없이 현재 리스트 결과를 재사용한다.

## 8. 펫동반 카페 / 공간 구조

이번 턴 구현 범위:

- 주변 펫동반 카페 / 공간 리스트
- 상세 진입
- 정적 지도 미리보기
- 외부 지도 열기

각 항목 최소 정보:

- 상호명
- 거리
- 카테고리
- 주소
- 좌표
- 펫동반 관련 안내 문구

중요:

- `실제 반려동물 동반 가능 여부`는 단순 Kakao 키워드 검색만으로는 보장할 수 없다.
- Kakao Local API 공식 문서 기준 응답 필드는 이름, 주소, 좌표, 카테고리, 장소 URL 등이며 `반려동물 동반 가능 여부` 전용 필드는 없다.
- 따라서 현재 구현은 UI 구조와 provider 분리까지만 열고, `검증된 반려동물 동반 데이터 소스` 연결 전에는 안전을 위해 결과를 바로 노출하지 않는 방향이 맞다.
- 실제 운영 단계에서는 한국관광공사 반려동물 동반여행 서비스나 지자체 공공데이터, 또는 운영 검수/CMS 기반 데이터셋을 연결해야 한다.
- 현재 메뉴명도 안전하게 `펫동반 음식점`보다 넓은 범위의 `펫동반 카페 / 공간`으로 정리한다.
- 현재 provider만으로는 `실제 장소 사진`도 안정적으로 제공받지 못하므로, 산책 장소 상세에서는 이미지 탭을 두지 않고 메타 정보와 지도/장소 링크 액션만 유지한다.
- 실제 장소 사진을 붙이려면 공원/장소별 공식 이미지 API, 운영 검수 이미지, 또는 별도 저작권 확인된 이미지 소스가 필요하다.

이후 확장 가능성:

- 카페 / 식당 필터
- 실내 / 실외
- 소형견 가능
- 동반 조건

## 9. 상태 처리 방식

각 화면은 아래 상태를 명확히 가진다.

- `idle`
- `location-loading`
- `search-loading`
- `refreshing`
- `ready`
- `empty`
- `error`

권한 상태는 별도 값으로 유지한다.

- `granted`
- `denied`
- `blocked`
- `unavailable`

즉, 위치 권한과 결과 로딩을 섞지 않는다.

## 11. 서비스 계층 구조

이번 기능은 UI에서 provider를 직접 다루지 않게 아래처럼 나눈다.

- 공용 위치/검색 요청 파라미터 normalize
- Kakao Local provider
- 산책 장소 검색 service
- 펫동반 음식점 검색 service
- 결과 normalize
- 거리/시간 포맷
- 지도 미리보기/외부 지도 링크 builder

즉, 화면은 `walk`와 `pet-friendly-place` 도메인을 알고, provider 구현은 몰라야 한다.

## 12. 이번 턴 범위

이번 턴에서 실제로 여는 범위:

- 전체메뉴 안 별도 메뉴 2개

- 검색 기반 조회
- 리스트 화면
- 상세 화면
- 정적 지도 미리보기
- 외부 지도 열기

이번 턴에서 넘기는 범위:

- 최근 검색어
- 자동완성
- 실제 경로 탐색
- 즐겨찾기
- 최근 본 장소
- 산책 기록 저장
- 고급 필터 UI
