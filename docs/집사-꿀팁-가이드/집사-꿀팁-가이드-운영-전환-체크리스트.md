# 집사 꿀팁 가이드 운영 전환 체크리스트

## 1. 문서 목적

이 문서는 집사 꿀팁 가이드가 `seed fallback 중심 QA 상태`에서 `실제 운영 공개 row 중심 상태`로 넘어갈 때 확인해야 하는 기준을 정리한 운영 전환 체크리스트다.

목표는 아래 두 가지다.

- seed와 운영 공개 row가 섞여 보이거나 잘못 해석되는 상황을 막는다.
- 운영 공개 row 전환 이후 홈 추천, 더보기 목록, 검색, 상세 흐름이 그대로 유지되는지 빠르게 확인한다.

## 2. 전환 전 확인

- `pet_care_guides`에 공개 row가 실제로 준비되어 있는지 확인한다.
- 공개 조건이 아래 기준을 만족하는지 확인한다.
  - `status = published`
  - `is_active = true`
  - `deleted_at is null`
  - `published_at is null or published_at <= now()`
- 대표 종별 최소 공개 row가 준비되어 있는지 확인한다.
  - 강아지
  - 고양이
  - 기타 대표 종 1개 이상
- 공통 팁 row도 최소 1개 이상 준비되어 있는지 확인한다.

## 3. 전환 직후 확인

- 홈 추천 source 배지가 `운영 데이터`로 바뀌는지 확인한다.
- 더보기 목록 source 배지가 `운영 데이터`로 바뀌는지 확인한다.
- `테스트 seed` 카드 배지가 사라지는지 확인한다.
- 검색 결과가 운영 공개 row 기준으로도 정상 동작하는지 확인한다.

## 4. 선택 펫 기준 확인

- 강아지 선택 시 강아지 + 공통 팁 중심으로 노출되는지 확인한다.
- 고양이 선택 시 고양이 + 공통 팁 중심으로 노출되는지 확인한다.
- 기타 대표 종 선택 시 해당 종 + 공통 팁 중심으로 노출되는지 확인한다.
- 다른 종 전용 팁이 더보기 기본 목록에 섞이지 않는지 확인한다.

## 5. 개인화 기준 확인

- 세부 종/품종을 바꾸면 홈 추천이 다시 계산되는지 확인한다.
- 생년월일 변경 후 어린 개체 / 성체 / 노령 분기가 바뀌는지 확인한다.
- memorial 상태 펫은 일반 케어 추천이 숨겨지는지 확인한다.

## 6. 운영 row 품질 확인

- 제목/요약/본문 미리보기/본문이 비어 있지 않은지 확인한다.
- 카테고리, 태그, `target_species`, `species_keywords`, `search_keywords`가 추천/검색에 쓸 수 있는 수준으로 채워졌는지 확인한다.
- `priority`, `sort_order`, `rotation_weight`가 극단값으로 들어가 있지 않은지 확인한다.
- 이미지가 있는 row는 썸네일/대표 이미지 URL이 깨지지 않는지 확인한다.

## 7. 전환 후 정책

- 운영 공개 row가 준비되면 QA는 seed가 아니라 운영 row 기준으로 본다.
- seed fallback은 운영 row가 비거나 원격 조회가 실패했을 때의 안전장치로만 유지한다.
- 운영 공개 row 검증이 끝난 뒤에는 배지 기준으로 seed 노출 여부를 반드시 같이 확인한다.

## 8. 최종 판정

아래를 모두 만족하면 운영 전환 완료로 본다.

- source가 `운영 데이터`로 안정적으로 유지된다.
- 홈 추천 / 더보기 / 검색 / 상세가 운영 공개 row 기준으로 정상 동작한다.
- 선택 펫 종 / 세부 종 / 나이 / memorial 분기가 그대로 유지된다.
- seed 카드가 더 이상 기본 QA 결과에 섞여 보이지 않는다.
