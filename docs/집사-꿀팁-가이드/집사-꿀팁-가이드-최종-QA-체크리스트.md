# 집사 꿀팁 가이드 최종 QA 체크리스트

## 1. 문서 목적

이 문서는 집사 꿀팁 가이드 기능을 실제 출시 전 기준으로 점검할 때 사용하는 최종 QA 체크리스트다.

목표는 아래 세 가지다.

- 운영 공개 row와 seed fallback 상태를 혼동하지 않고 점검한다.
- 홈 추천, 더보기 목록, 검색, 상세, 관리자 브릿지 흐름을 한 번에 확인한다.
- 펫 기반 개인화, memorial 분기, 이벤트 적재, 공개 데이터 전환 기준을 빠짐없이 본다.

## 2. 사전 확인

- 현재 빌드가 최신 코드 기준인지 확인한다.
- Supabase 공개 row가 있는지 확인한다.
- 공개 row가 없으면 seed fallback 정책이 동작하는지 확인한다.
- 테스트 계정에 아래 케이스가 준비되어 있는지 확인한다.
  - 강아지
  - 고양이
  - 기타 대표 종 1종 이상
  - 생년월일이 있는 펫
  - 생년월일이 없는 펫
  - memorial 상태 펫

## 3. 데이터 소스 확인

### 3.1 source 배지 확인

- 홈 추천과 가이드 목록에서 현재 source 배지가 보이는지 확인한다.
- source 의미:
  - `운영 데이터`: 원격 공개 row 사용
  - `테스트 seed`: seed fallback 사용
  - `원격 빈 상태`: 원격 조회는 성공했지만 공개 row 0건

### 3.2 seed fallback 정책 확인

- 실제 공개 데이터가 없으면 seed가 자동으로 노출되는지 확인한다.
- 홈 추천, 더보기 목록, 검색, 상세가 모두 seed 기준으로도 깨지지 않는지 본다.
- seed 카드는 개발 환경에서 `테스트 seed` 배지가 보이는지 확인한다.

## 4. 홈 추천 QA

### 4.1 선택 펫 반영

- 선택 펫을 강아지로 바꾸면 강아지 + 공통 팁 중심으로 바뀌는지 확인한다.
- 선택 펫을 고양이로 바꾸면 고양이 + 공통 팁 중심으로 바뀌는지 확인한다.
- 선택 펫을 기타 대표 종으로 바꾸면 해당 종 + 공통 팁 중심으로 바뀌는지 확인한다.

### 4.2 세부 종 반영

- 같은 대표 종 안에서 세부 종/품종을 수정하면 홈 추천이 다시 계산되는지 확인한다.
- 더보기 목록과 홈 추천이 같은 세부 종 기준으로 정합성을 유지하는지 확인한다.

### 4.3 나이/생애주기 반영

- 어린 개체는 어린 개체성 팁이 우선 노출되는지 확인한다.
- 성체는 공통/성체성 팁이 자연스럽게 섞이는지 확인한다.
- 노령 개체는 노령성 팁이 우선 노출되는지 확인한다.

### 4.4 memorial 분기

- `death_date`가 있는 펫은 일반 케어 추천이 숨겨지는지 확인한다.
- memorial 상태 문구가 추억/기록 중심으로 바뀌는지 확인한다.
- memorial 펫에서 list/detail 클릭 이벤트 메타데이터에 `isMemorialPet=true`가 남는지 확인한다.

## 5. 더보기 목록 QA

- 기본 목록이 선택 펫 종 + 공통 팁만 노출하는지 확인한다.
- 다른 종 전용 팁이 기본 목록에 섞이지 않는지 확인한다.
- 검색을 하지 않은 기본 목록과 검색 결과 목록이 같은 공개 조건을 따르는지 확인한다.
- 빈 상태 문구가 현재 source에 맞게 자연스럽게 보이는지 확인한다.

## 6. 검색 QA

- 최근 검색어가 저장/노출/삭제 흐름을 유지하는지 확인한다.
- 인기 검색어가 정상 노출되는지 확인한다.
- 검색 결과 클릭 시 상세로 이동이 깨지지 않는지 확인한다.
- 검색 source에 따라 이벤트 메타데이터가 달라지는지 확인한다.
  - `guide-list`
  - `guide-search`

## 7. 상세 QA

- 목록/홈/검색에서 상세 진입이 모두 정상 동작하는지 확인한다.
- 상세 본문과 메타 정보가 seed/운영 row 모두에서 안정적으로 렌더링되는지 확인한다.
- 뒤로가기 후 목록 상태와 선택 펫 컨텍스트가 유지되는지 확인한다.

## 8. 관리자 브릿지 QA

- 관리자 권한 계정만 `가이드 운영` 진입이 가능한지 확인한다.
- 일반 사용자 계정에서는 관리자 진입이 보이지 않거나 차단되는지 확인한다.
- 등록/수정/발행/보관/비활성화 후 공개 조회 흐름이 깨지지 않는지 확인한다.

## 9. 이벤트/리포트 QA

- 홈 추천 노출 이벤트가 남는지 확인한다.
- 목록 클릭 이벤트가 남는지 확인한다.
- 상세 조회 이벤트가 남는지 확인한다.
- 이벤트 메타데이터에 아래 값이 포함되는지 확인한다.
  - `strategyVersion`
  - `source`
  - `guideCategory`
  - `guideTargetSpecies`
  - `petRepresentativeSpecies`
  - `isMemorialPet`
  - `seasonBucket`

리포트 SQL 참고:

- `docs/sql/도메인/집사-꿀팁-가이드/집사-꿀팁-가이드-이벤트-리포트-점검.sql`

## 10. seed -> remote 전환 QA

- 운영 공개 row를 넣은 뒤 source가 `운영 데이터`로 바뀌는지 확인한다.
- 운영 공개 row를 제거하면 다시 seed fallback으로 내려오는지 확인한다.
- seed와 remote가 섞여 보이지 않는지 확인한다.
- 운영 row 준비 후에는 seed 배지가 사라지는지 확인한다.

## 11. 실기기 UX QA

- 홈 추천 카드 스크롤과 탭 반응이 자연스러운지 확인한다.
- 더보기 목록 검색창에서 키보드가 올라와도 스크롤과 탭이 자연스러운지 확인한다.
- 반려동물 프로필 수정에서 세부 종/몸무게 입력이 키보드에 가려지지 않는지 확인한다.
- 입력 필드 포커스 이동과 저장 버튼 가시성이 iOS/Android 모두 자연스러운지 확인한다.

## 12. 최종 판정 기준

아래를 모두 만족하면 앱 기준 집사 꿀팁 가이드는 출시 직전 QA 통과 상태로 본다.

- 홈 추천 / 더보기 / 검색 / 상세가 모두 정상 동작
- 선택 펫 종 / 세부 종 / 나이 / memorial 분기가 모두 반영
- 운영 row와 seed fallback 상태를 혼동 없이 구분 가능
- 이벤트 적재와 리포트 점검 SQL이 동작
- 관리자 브릿지와 사용자 소비 흐름이 서로 깨지지 않음
- 실기기 기준 키보드/스크롤/포커스 UX가 안정적임
