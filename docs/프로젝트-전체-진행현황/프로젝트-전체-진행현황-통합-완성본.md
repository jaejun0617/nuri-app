# 프로젝트 전체 진행현황 통합 완성본

## 1. 문서 목적

이 문서는 NURI 프로젝트 전체의 현재 진행률과 남은 작업을 한 번에 볼 수 있도록 정리한 기준 문서다.

목표는 아래 세 가지다.

- 앱과 웹의 현재 위치를 분리해서 본다.
- 도메인별 완료 범위와 남은 마감 작업을 한 번에 파악한다.
- 이후 작업 우선순위를 흔들리지 않게 유지한다.

## 2. 현재 전체 진행률

현재 기준 전체 프로젝트 진행률은 아래로 본다.

- 전체 프로젝트: `86~89%`
- 앱 `nuri`: `85~89%`
- 웹 `nuri-web`: `55~60%`

판단 기준:

- 앱은 핵심 사용자 기능과 주요 도메인 구조가 대부분 열려 있고, 지금은 운영 완성도와 출시 마감 단계에 가깝다.
- 웹은 챕터 1~3 기반 구조는 열렸지만, 현재 우선순위는 앱 완성 쪽이므로 당분간 보류한다.

## 3. 앱 기준 진행현황

### 3.1 전체 앱 진행률

- 앱 전체: `85~89%`

### 3.2 핵심 도메인별 상태

- 인증 / 프로필 / 기본 홈 흐름: 완료에 가까움
- 멀티펫 / 펫 등록 / 펫 수정: 완료에 가까움
- 타임라인 / memory_images 구조: 핵심 범위 완료
- 집사 꿀팁 가이드: `93~96%`
- 출시 전 QA / 운영 마감: 진행 중

## 4. 집사 꿀팁 가이드 상태

### 4.1 챕터 기준 상태

- 챕터 1 완료
- 챕터 2-1 완료
- 챕터 2-2 완료
- 챕터 3-1 완료
- 챕터 3-2 완료
- 챕터 3-3 완료

즉, 집사 꿀팁 가이드는 새 기능 챕터보다 `운영 전환 / 최종 QA / 출시 마감 단계`에 들어와 있다.

### 4.2 현재 완료된 핵심 범위

- 홈 추천 / 더보기 목록 / 상세 / 검색
- 최근 검색어 / 인기 검색어
- seed fallback 정책
- 선택 펫 종 + 공통 팁 기준 목록 노출
- 세부 종 / 나이 / memorial 분기
- 추천 가중치 / 계절성 / 반복 노출 완화
- 이벤트 적재 / 리포트 SQL 점검 기준
- 관리자 브릿지 최소 CMS
- QA 체크리스트 / 운영 전환 체크리스트

### 4.3 남은 마감 작업

- 운영 공개 row 기준 최종 검증
- 실기기 최종 QA 실제 수행
- 추천 문구와 체감 튜닝 마지막 보정
- 필요 시 운영 리포트 화면 초안

## 5. 타임라인 메모리 이미지 시스템 상태

현재 상태:

- 구조 전환 핵심 범위 완료
- `memory_images` 전환 완료
- worker / backfill / fallback / preload 최적화 기준 정리 완료
- 운영 관찰 포인트 문서화 완료

남은 작업:

- 실제 운영 관찰 결과 기반 소규모 보정
- 출시 전 QA와 성능 점검 마감

## 6. 웹 CMS 상태

### 6.1 현재 진행률

- `nuri-web`: `55~60%`

### 6.2 현재 상태

- 챕터 1 완료
  - 공개 메인 / 로그인 / 관리자 읽기 구조
- 챕터 2 완료
  - 등록 / 수정 / 상태 변경
- 챕터 3 완료
  - 이미지 업로드 / 미리보기 / 관리자 UI 정리

### 6.3 현재 판단

- 웹 CMS는 구조적으로 시작은 잘 열렸다.
- 다만 현재 전체 프로젝트 우선순위는 앱 완성이다.
- 따라서 웹은 당분간 기능 확장보다 보류/유지 상태로 본다.

## 7. 지금 남은 최우선 작업

현재 우선순위는 아래 순서가 맞다.

1. 앱 운영 공개 row 기준 최종 검증
2. 앱 실기기 최종 QA
3. 집사 꿀팁 가이드 마지막 운영 마감
4. 타임라인/기록 도메인 출시 전 QA
5. 전체 프로젝트 문서/운영 기준 최종 정리

## 8. 99% 기준 남은 작업

프로젝트를 `99%`에 가깝게 보려면 아래가 필요하다.

- 앱 기준 공개 데이터와 seed fallback 전환 검증 완료
- 홈/검색/상세/프로필 수정 실기기 QA 완료
- 운영/테스트 데이터 혼동 없는 최종 정책 확정
- 주요 도메인 빈 상태/에러 상태/문구 마감
- 출시 직전 체크리스트 기준 패스

## 10. 결론

현재 프로젝트는 기능을 새로 여는 단계보다, `실제 운영과 출시를 위해 마감 품질을 올리는 단계`에 들어와 있다.

특히 앱 기준으로는:

- 큰 구조는 거의 열렸고
- 남은 것은 운영 전환, QA, 문구/체감 보정, 출시 마감이다.

즉 지금부터의 작업은 `무엇을 더 만들지`보다 `무엇을 더 안전하게 마감할지`가 중심이다.
