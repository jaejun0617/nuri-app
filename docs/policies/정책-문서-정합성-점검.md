## 2. 현재 구현 기준으로 문서에 확정 기재 가능한 항목

- 이메일 기반 회원가입, 로그인, 비밀번호 재설정이 존재한다.
- 회원가입 시 `이용약관`, `개인정보처리방침`은 필수 동의, `마케팅 수신 동의`는 선택 동의다.
- 운영 주체 표기는 현재 `펫 누리(Pet Nuri) 운영자 신재준`으로 반영됐다.
- 공식 문의 채널은 현재 `pet_nuri@naver.com`이다.
- 동의 이력은 `user_consent_history` 계약으로 저장된다.
- 계정 삭제는 `요청 접수`와 `최종 정리 완료`를 분리해 처리한다.
- 계정 삭제와 연동된 cleanup queue 계약이 있다.
- 계정 삭제 후 동일 이메일 재사용은 허용 방향으로 문서화됐지만, 실제 재가입 가능 시점은 운영 정책과 처리 완료 상태에 따라 달라질 수 있도록 보수적으로 적었다.
- 반려동물 프로필, 기록, 기록 이미지, 일정, 가이드, 장소/여행, 커뮤니티 기능이 실제 코드에 존재한다.
- 커뮤니티는 신고, 자동 숨김, moderation queue, action log, reporter trace, 이미지 cleanup trace를 운영 기준으로 다룬다.
- 커뮤니티 정책에는 `경고 / 일시 정지 / 영구 이용 정지`의 제재 방향과 `pet_nuri@naver.com` 이의 제기 채널이 반영됐다.
- 위치 기능은 기기 좌표를 사용하며, 마지막 좌표 일부를 로컬 캐시에 저장할 수 있다.
- 위치/날씨/장소 기능은 외부 API와 연동된다.
- 모니터링은 Sentry, Firebase Crashlytics가 코드상 존재한다.
- 공개 문서 기준 보관 원칙은 `탈퇴 시 지체 없이 파기`, `부정이용 방지 목적 비식별화 후 1년`, `로그인 기록 3개월`로 정리됐다.
- 마케팅 정보 수신은 `마케팅-정보-수신-동의서.md` 독립 문서로 분리됐고, 공개 대상 문서로 분류됐다.
- 개인정보처리방침에는 `Supabase, Inc.`와 `Sentry`를 `처리위탁 + 국외 이전` 모두 명시하는 방향이 반영됐다.

## 6. 현재 코드와 문서가 어긋날 가능성이 있는 항목

- 회원가입 화면과 More의 계정 삭제 안내는 이미 실제 외부 링크를 연다.
- `LEGAL_DOCUMENTS`의 마케팅 draftPath는 여전히 기존 `마케팅-수신-안내-초안.md`를 가리키고 있고, 공개 대상 기준으로 잠긴 `마케팅-정보-수신-동의서.md`와의 코드 정렬은 다음 허용 턴의 후속이다.
- 계정 삭제 cleanup worker RPC follow-up이 남아 있으므로, 삭제 후 모든 cleanup이 항상 즉시 닫힌다고 적으면 안 된다.
- 커뮤니티 restore / resolved 경로는 remote follow-up이 남아 있으므로, 자동 숨김 복구가 운영에서 완전히 닫혔다고 적으면 안 된다.
- 장소/여행 후보 정보는 공개 신뢰 정보와 동일한 수준으로 확정된 것이 아니다.
- 공개 문서 기준 보관기간은 운영 정책으로 정리됐지만, 기간 집행 automation이나 만료 purge 절차를 이미 구현된 것처럼 쓰면 안 된다.
