# 커뮤니티 목록 화면 UI 명세

## 1. 목적

- 커뮤니티 목록 화면은 `가볍게 훑는 피드` 역할을 맡는다.
- 실제 콘텐츠 집중은 상세 화면에서 처리한다.
- 목록은 렌더링 비용을 낮추고, 텍스트 중심으로 빠르게 스캔할 수 있어야 한다.

## 2. 헤더 규칙

- 좌측: 뒤로가기 화살표
- 중앙: `커뮤니티`
- 우측: 원형 `+` 버튼
- 우측 `+` 버튼은 기존 타임라인/일정 화면에서 사용하는 공통 원형 헤더 액션 패턴을 그대로 따른다.
- 1단계에서는 작성 화면이 아직 없으므로 `+` 버튼을 눌렀을 때 준비 중 토스트를 노출한다.

## 3. 목록 카드 규칙

- 카드 전체를 눌러 상세 화면으로 이동한다.
- 카드 안에는 별도 액션 버튼을 두지 않는다.
- 카드 높이는 가능한 한 일정하게 유지한다.
- 카드에 노출할 정보:
  - 작성자 닉네임
  - 반려동물 정보
    - `petName · petSpecies`
    - 값이 없으면 숨김
  - 카테고리 배지
  - `이미지` 배지
    - 게시글에 이미지가 있을 때만 노출
  - 본문
    - `numberOfLines={2}` 기준
  - 좋아요 수
  - 댓글 수
  - 상대 시간

## 4. 이미지 노출 정책

- 목록에서는 실제 이미지를 렌더링하지 않는다.
- 목록에서는 이미지 존재 여부만 `이미지` 배지로 힌트 제공한다.
- 상세 화면에서만 실제 이미지를 렌더링한다.
- 이 정책은 스크롤 성능, 카드 높이 안정성, 텍스트 중심 커뮤니티 톤 유지를 위한 기본 원칙이다.

## 5. 카테고리 탭 규칙

- 상단에 카테고리 칩을 둔다.
- 기본 카테고리:
  - `전체`
  - `질문`
  - `팁 공유`
  - `일상`
  - `정보`
- 현재 선택된 반려동물 테마 색을 active 상태에 사용한다.
- 카테고리 변경 시:
  - 목록을 새로 조회한다.
  - 스크롤 위치를 맨 위로 올린다.

## 6. 페이지네이션 규칙

- 페이지네이션은 cursor 기반 무한 스크롤을 사용한다.
- `onEndReached` 시 다음 페이지를 로드한다.
- 다음 페이지 로드 시 자동으로 맨 위로 이동하지 않는다.
- 사용자가 보고 있던 문맥을 유지하는 것이 우선이다.
- 목록 상단 이동은 사용자가 직접 탑 버튼을 누를 때만 수행한다.

## 7. 탑 버튼 규칙

- 일정 스크롤 이상 내려갔을 때만 우하단에 노출한다.
- 원형 버튼으로 노출한다.
- 현재 선택된 반려동물 테마 색을 사용한다.
- 누르면 `scrollToOffset({ offset: 0, animated: true })`로 이동한다.

## 8. 상태별 UI 규칙

- 초기 로딩:
  - 중앙 로딩 인디케이터
- 에러:
  - 짧은 안내 문구
  - `다시 시도` 버튼
- 빈 상태:
  - `아직 게시글이 없어요. 첫 번째로 공유해 보세요!`
  - 1단계에서는 작성 화면이 아직 없으므로 버튼은 준비 중 토스트를 띄운다.
- 더 불러오는 중:
  - 리스트 footer에 로딩 인디케이터

## 9. 상세 화면 규칙

- 목록에서 누른 카드의 전체 본문을 보여준다.
- 실제 이미지는 상세 화면에서만 노출한다.
- 댓글 목록은 상세 화면에서 읽기 전용으로 먼저 연다.
- 1단계에서는 댓글 입력, 좋아요, 신고, 작성, 수정 흐름을 붙이지 않는다.

## 10. 1단계 구현 범위와 연결

- 포함:
  - 목록 조회
  - 상세 조회
  - 댓글 조회
  - More 메뉴 진입
  - 읽기 상태 store
  - 커스텀 헤더
  - 무한 스크롤
  - 탑 버튼
- 제외:
  - 게시글 작성
  - 게시글 수정
  - 댓글 작성
  - 좋아요 액션
  - 신고
  - 이미지 업로드

## 11. 실서비스 기준 주의점

- 목록 카드에서 이미지를 렌더링하지 않는 정책을 유지한다.
- 무한 스크롤과 카테고리 전환은 서로 다른 UX 규칙을 가진다.
  - 다음 페이지 로드: 현재 위치 유지
  - 카테고리 변경: 맨 위로 이동
- 렌더링 최적화를 위해 카드 props는 최소화한다.
- 리스트는 `FlatList` 기준으로 구현하고, 카드 컴포넌트는 `memo`를 기본 적용한다.
