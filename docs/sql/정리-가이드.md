# docs/sql 정리 가이드

`docs/sql`은 프로젝트의 모든 SQL을 한 번에 보는 기준 폴더다. 도메인 SQL, 공용 릴리즈 SQL, migration을 여기서 역할별로 나눈다.

설계/운영 설명 문서는 `docs/domains/`와 `docs/operations/`에서 관리하고, 실행하거나 검토할 SQL은 `docs/sql/` 아래에서 바로 찾을 수 있게 유지한다.

- `docs/operations/마이그레이션-운영-기준/마이그레이션-운영-기준-통합-완성본.md`
- `docs/operations/공용-릴리즈-묶음-운영-기준/공용-릴리즈-묶음-운영-기준-통합-완성본.md`

## 현재 폴더 구조

- `docs/sql/도메인/`
  - 도메인별 운영 SQL, 초안 SQL, seed SQL
  - 예: 집사 꿀팁 가이드, 타임라인, 펫동반 장소 탐색
- `docs/sql/마이그레이션/`
  - 실제 배포 이력 기준 migration 파일
  - 날짜 prefix를 유지하고, 배포 순서 추적이 가능해야 한다
- `docs/sql/공용-릴리즈-묶음/전체-초기-구성/`
  - 새 Supabase 프로젝트를 한 번에 세팅할 때 참고하는 전체 초기 구성 SQL
- `docs/sql/공용-릴리즈-묶음/기능-추가/`
  - 운영 중 특정 기능을 별도로 추가할 때 쓰는 수동 실행 SQL

## 문서 운용 규칙

- SQL은 도메인 구분과 관계없이 `docs/sql/` 아래에 모은다.
- 도메인 기준 SQL은 `docs/sql/도메인/<도메인명>/`에서 관리한다.
- 같은 도메인의 SQL도 `최종본 / 초안 / seed`를 이 폴더 안에서 같이 본다.
- 새 파일을 추가할 때는 `도메인 최종본인지`, `migration인지`, `수동 릴리즈 묶음인지` 먼저 구분한다.
- 경로만 보고 역할을 이해할 수 있게 폴더를 먼저 나눈다.
- 처음 보는 사람도 `docs/sql`만 열면 SQL 자산 위치를 바로 파악할 수 있어야 한다.
