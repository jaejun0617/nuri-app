# docs/sql 정리 가이드

`docs/sql`은 프로젝트의 모든 SQL을 모아 관리하는 기준 폴더다. 도메인 최종본, 배포 migration, 공용 릴리즈 SQL을 이 안에서 역할별로 나눈다.

설명 문서는 `docs` 루트의 작업 폴더에서 같이 관리한다.

- `docs/마이그레이션-운영-기준/마이그레이션-운영-기준-통합-완성본.md`
- `docs/공용-릴리즈-묶음-운영-기준/공용-릴리즈-묶음-운영-기준-통합-완성본.md`

## 현재 폴더 구조

- `docs/sql/도메인/`
  - 집사 꿀팁 가이드, 타임라인 같은 도메인 기준 최종본 SQL
  - 실무에서 바로 참고할 canonical SQL은 이 경로를 먼저 본다
- `docs/sql/마이그레이션/`
  - 실제 배포 이력 기준 migration 파일
  - 날짜 prefix를 유지하고, 배포 순서 추적이 가능해야 한다
- `docs/sql/공용-릴리즈-묶음/전체-초기-구성/`
  - 새 Supabase 프로젝트를 한 번에 세팅할 때 참고하는 전체 초기 구성 SQL
- `docs/sql/공용-릴리즈-묶음/기능-추가/`
  - 운영 중 특정 기능을 별도로 추가할 때 쓰는 수동 실행 SQL

## 문서 운용 규칙

- 집사 꿀팁 가이드, 타임라인 같은 도메인 SQL 최종본도 이제 `docs/sql/도메인/` 아래로 모은다.
- 새 파일을 추가할 때는 `도메인 최종본인지`, `migration인지`, `수동 릴리즈 묶음인지` 먼저 구분한다.
- 경로만 보고 역할을 이해할 수 있게 폴더를 먼저 나눈다.
