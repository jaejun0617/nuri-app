# 산책 조회 신뢰성 개선 통합 완성본

## 1. 현재 정책

- 산책 조회 기준은 `현재 GPS 기반 주변 추천 + 직접 검색`으로 고정한다.
- `다른 동 선택`, `고양시 전체` 같은 근사치 지역 확장 기능은 제거한다.
- 거리 기준은 항상 `현재 위치 기준`이다.
- 검색은 현재 위치가 없어도 허용하되, 거리값은 `거리 확인 중`으로 처리한다.

## 2. 조회 기준 단일화 방식

- 기본 진입:
  - 현재 GPS 좌표를 anchor로 사용한다.
  - `공원`, `문화공원`, `호수공원`, `근린공원`, `생태공원`, `수변공원`, `어린이공원`, `산책로`, `둘레길`, `숲길` 키워드로 주변 추천을 가져온다.
- 검색:
  - 사용자가 입력한 검색어를 그대로 Kakao keyword search에 사용한다.
  - 현재 위치가 있으면 현재 위치 주변 검색으로, 없으면 텍스트 검색으로 동작한다.
- 거리:
  - 현재 위치가 있으면 현재 위치 기준 거리(m)를 계산한다.
  - 현재 위치가 없으면 거리값은 null이며, UI에는 `거리 확인 중`으로 표시한다.

## 3. 제거한 구조

- selected district scope 제거
- city scope 제거
- strict district filter 제거
- preferred district ranking 제거
- 동 선택 칩 UI 제거
- `고양시 전체` UI 제거

## 4. 유지한 구조

- 리스트 결과 컬렉션을 상세로 전달하는 구조
- 상세 연관 장소 `더보기`
- 외부 지도 열기 / 장소 링크
- 추천순 / 가까운순 / 멀리순 정렬
- 검색 입력 UX 개선

## 5. 키보드 바 + 스크롤 안정성 대응

- 검색바가 있는 리스트 화면은 `KeyboardAvoidingView`로 감싼다.
- 리스트에는 아래 속성을 유지한다.
  - `keyboardShouldPersistTaps="handled"`
  - `keyboardDismissMode`
  - `automaticallyAdjustKeyboardInsets`
- 검색 입력은 `blurOnSubmit={false}`를 유지해 키보드 상단 예측 바가 있는 상태에서도 입력 흐름을 안정적으로 유지한다.

## 6. 현재 한계

- 현재 위치가 없는 상태의 직접 검색은 장소는 보여줄 수 있지만 거리값은 보여줄 수 없다.
- Kakao Local keyword search만으로는 운영 검수된 공원 데이터셋 수준의 정확도를 100% 보장하지는 못한다.

## 7. 남은 확장 포인트

- 운영 검수 POI 데이터셋 연결
- 최근 검색어 / 즐겨찾기 / 길찾기 연동
- 산책 기록 저장
