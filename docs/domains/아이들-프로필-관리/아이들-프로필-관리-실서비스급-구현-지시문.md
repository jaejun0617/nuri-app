# 아이들 프로필 관리 — NURI 실서비스급 구현 지시문

## 목적

- 멀티펫 기준의 `아이들 프로필 관리` 허브 화면을 신설한다.
- 현재 선택된 아이의 `프로필 수정`으로 바로 들어가는 구조를 `관리 허브 -> 개별 수정` 구조로 전환한다.
- 렌더링 성능, 상태 정합성, 유지보수성을 우선한다.

## 우선순위

1. 렌더링 성능
2. 코드 유지보수성
3. UX 완성도
4. 구현 속도

## 현재 코드 기준 핵심 원칙

- 펫 상태 source of truth는 `src/store/petStore.ts`
- 펫 이미지는 현재 `public bucket + avatarPath + avatarUrl(publicUrl)` 구조를 유지한다
- signed URL 캐시 레이어는 이번 단계에 만들지 않는다
- 관리 허브 화면은 목록/선택/수정 진입 역할만 담당한다
- 삭제, 정렬, 드래그, 과한 카드 액션은 넣지 않는다

## 스토어 규칙

- 기존 `Pet` 타입 필드 이름을 바꾸지 않는다
- 아래 action만 추가한다
  - `invalidatePetAvatar(petId, options?)`
- 역할:
  - 편집 후 avatarPath가 바뀐 경우 기존 avatarUrl을 무효화한다
  - 이후 `fetchMyPets -> setPets` 흐름으로 최신 public URL을 다시 반영한다

## 이미지 처리 규칙

- 관리 화면은 `pet.avatarUrl`만 사용한다
- 이미지가 없으면 fallback 렌더
- 작은 고정 썸네일만 사용한다
- 카드가 이미지 URL을 자체적으로 재계산하지 않는다
- 성능 핵심은 이미지 캐시 레이어보다 `리스트 리렌더 최소화`다

## 카드 규칙

- 카드에는 아래만 표시한다
  - 아바타
  - 이름
  - `품종 · 나이`
  - 추모 배지
  - `이 아이로 보기`
  - `프로필 수정`
- `React.memo`와 custom comparator를 사용한다
- 카드 내부에서 store를 직접 구독하지 않는다

## 화면 규칙

- `FlatList` 사용
- `usePetStore`는 selector 분리 사용
- `selectedPetId` 전환은 즉시 반영
- 전체 화면 loading overlay는 두지 않는다
- Empty 상태에서는 `아이 추가하기` CTA를 제공한다

## 네비게이션 규칙

- More > `아이들 프로필 관리`는 `PetManagement` 라우트로 연결
- `PetManagement`에서만 `PetProfileEdit`로 들어간다
- `entrySource: 'more'`를 유지해 뒤로가기가 More 컨텍스트를 잃지 않게 한다

## 프로필 수정 정합성 규칙

- `PetProfileEditScreen` 저장 완료 흐름은 기존 완료 UX를 유지한다
- avatarPath가 바뀌면 `invalidatePetAvatar(pet.id)`를 먼저 호출한다
- 그 후 기존처럼 `fetchMyPets()`와 `setPets()`를 실행한다

## QA 체크포인트

- More에서 관리 허브 화면이 열린다
- 등록된 모든 펫이 카드로 보인다
- 선택된 펫에 `지금 보고 있는 아이` 배지가 붙는다
- `이 아이로 보기` 탭 시 `selectedPetId`가 즉시 바뀐다
- `프로필 수정` 탭 시 정확한 `petId`로 이동한다
- 수정 후 복귀 시 이름/이미지/메타가 최신 상태다
- 새 아이 추가 후 리스트에 즉시 반영된다
- 이미지 없는 펫 fallback이 정상 노출된다
- 펫 1마리 / 3마리 / 5마리에서도 스크롤이 흔들리지 않는다

## 절대 하지 말 것

- signed URL 캐시 신규 도입
- store와 별개인 avatar map 상태 추가
- `usePetStore()` 전체 구독
- 카드 안 액션 3개 이상 추가
- 삭제 기능 구현
- 화면 진입마다 무조건 서버 재조회

## 성공 기준

- `아이들 프로필 관리`가 실제 멀티펫 관리 허브로 동작한다
- 기존 `프로필 수정`은 개별 편집 화면 역할로 분리된다
- 리스트 렌더링과 이미지 표시가 안정적이다
- 홈/기록/일정 컨텍스트는 계속 `selectedPetId` 기준으로 자연스럽게 동작한다
