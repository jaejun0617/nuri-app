# 반려동물과 여행 초기 설계

## 1. 왜 별도 도메인으로 보는가

`펫동반 장소 탐색`은 생활권 안에서 바로 이동할 수 있는 현재 위치 중심 도메인이다.

`반려동물과 여행`은 아래 특성이 달라 별도 도메인으로 보는 편이 맞다.

- 관광지
- 숙소
- 문화시설
- 야외 공간
- 여행지

즉 같은 반려동물 동반 탐색이라도 사용자의 의도, 데이터 소스, 정렬 기준, 캐시 전략이 다르다.

## 2. 권장 도메인 분리

- `펫동반 장소 탐색`
  - 현재 위치 기반 일상 탐색
  - 카페 / 식당 / 실내 공간 / 야외 공간
  - Kakao Local 1차 후보 수집 중심
- `반려동물과 여행`
  - 여행/관광 목적 탐색
  - 관광지 / 숙소 / 문화시설 / 야외 여행지
  - 한국관광공사 반려동물 동반여행 서비스와 공공데이터 중심

## 3. 데이터 소스 권장 조합

- 일상 탐색
  - Kakao Local = 1차 후보 수집
  - Google Places = 상세 속성 보강
  - Supabase = 검증 상태 / 운영 메타 source of truth
- 여행 탐색
  - 관광공사 반려동물 동반여행 서비스 = 핵심 후보 소스
  - 공공데이터포털 지자체 데이터 = 보강
  - Kakao Local = 좌표/주변 보조 후보
  - Supabase = 운영 메타 source of truth

## 4. 현재 앱 구조와의 연결

현재 `src/services/locationDiscovery/*` 레이어는 일상 탐색에 더 잘 맞는다.

여행 도메인은 장기적으로 아래처럼 분리하는 것이 안전하다.

- 공통
  - 위치 권한
  - 지도 열기
  - 상세 화면 공통 레이아웃 일부
- 분리
  - provider
  - normalize
  - 추천 정렬
  - 검증 근거 표시
  - 카테고리 체계

## 5. 다음 구현 전제

- `pet_place_search_logs.source_domain`은 `pet-friendly-place`와 `pet-travel`을 모두 받을 수 있게 설계한다.
- Supabase canonical 메타는 공용으로 유지하되 source link와 external signal이 여행 소스까지 수용해야 한다.
- 여행 도메인은 런타임 직접 호출보다 배치 적재 + 메타 merge 비중이 더 커야 한다.
