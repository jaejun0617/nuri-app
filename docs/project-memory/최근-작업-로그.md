### 무엇을 바꿨는가

- 펫동반 장소 아이템에 `source`, `verification`, `petPolicy`를 분리한 상태 모델을 넣었다.
- Kakao raw 응답을 바로 UI에 넣지 않고, normalize 후보와 서비스 메타 병합 단계를 분리했다.
- `placeMeta.ts`를 추가해 향후 Supabase 장소 메타 병합 경계를 먼저 만들었다.
- `placeMeta.ts`에 canonical meta / source link / external signal 기준의 미래 메타 계약을 정리했다.
- 리스트/상세에서 출처, 검증 상태, 검증 근거, 펫동반 안내를 함께 노출하도록 바꿨다.
- 펫동반 리스트 상단 배너를 `외부 후보 + 서비스 검증 상태 분리` 정책에 맞게 수정했다.
- 문서와 메모리 문서를 현재 구현에 맞게 다시 정리했다.
- Kakao 외 보강 소스로 Google Places와 관광공사/공공데이터의 역할을 문서 기준으로 검토했다.
- 실제 서비스급 Supabase 장소 메타/RLS/트리거/인덱스 SQL을 canonical meta + source link + external signal 구조로 작성했다.
- Google Places와 관광공사/공공데이터를 실제 서비스에서 어떤 순서와 조건으로 호출할지 운영 전략 문서를 추가했다.
- `반려동물과 여행` 도메인을 장기적으로 분리하는 초기 설계를 문서로 남겼다.
- `docs` 루트는 역할 폴더만 남기고, SQL은 다시 `docs/sql` 아래로 한 번에 모이도록 정리 기준을 고정했다.

### 왜 바꿨는가

- Kakao Local은 장소 후보 수집 레이어일 뿐, 펫동반 여부의 확정 source of truth가 아니기 때문이다.
- `external-unverified` 한 단계만으로는 추정/제보/운영검수를 구분할 수 없었다.
- 이번 턴 목표가 메뉴 추가가 아니라 실제 서비스 구조를 여는 것이었기 때문이다.

### 남은 리스크

- 아직 Supabase 장소 메타 테이블이 없어 `admin-verified`, `user-reported` 같은 상태는 실제 운영 데이터로 채워지지 않는다.
- 실기기에서 키보드 예측 바, stale -> fresh 전환, 외부 지도 deep link를 추가 QA 해야 한다.
- Google Places는 보조 신호로 유효하지만 비용과 FieldMask 통제가 선행돼야 한다.
- 관광공사/공공데이터는 런타임보다 배치 적재가 더 맞다.
