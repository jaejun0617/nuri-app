- `source_links`는 이번 턴 범위에서 의도적으로 제외하고, `places.primary_source + primary_source_place_id`로 bootstrap하는 구조를 문서와 SQL에 반영했다.
- SQL 초안에 `updated_at` trigger, check constraint, index, RLS, admin/user write 범위 초안을 포함했다.
- 현재 타입에 DB 스키마 대응용 source/report/policy 타입을 추가해 문서와 타입 방향이 어긋나지 않게 정리했다.

### 왜 바꿨는가

- candidate-layer만으로는 `confirmed`를 열 수 없고, 실제 서비스 운영에서는 정책 근거와 유저 제보를 별도 저장소에 누적해야 하기 때문이다.
- 이번 턴 목표가 UI 고도화가 아니라 Trust Layer의 실제 기반을 만드는 것이기 때문이다.

### 검증 결과

### 남은 리스크

- SQL은 아직 Supabase에 적용하지 않았고, admin review 입력 플로우와 Aggregation Layer merge 규칙은 후속 작업이 필요하다.
- `source_links`를 뒤로 미뤘기 때문에 다중 provider canonical merge는 다음 단계에서 별도 확장이 필요하다.
