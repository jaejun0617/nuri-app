# 다음 작업 우선순위

## 지금 당장 할 것

### 1. 장소/여행/산책 물리 실기기 최종 QA

- 이유
  - 현재 남은 가장 큰 출시 리스크다.
  - 공개 라벨, `내 상태`, stale/conflict 문구, 지도 미리보기, 외부 지도 전환, 긴 설명 `더보기/접기`를 실제 기기 기준으로 최종 캡처해야 한다.
- 선행 조건
  - Android 실기기 `SM_S937N`
  - 위치 권한 허용
  - PO 동행 확인

### 2. 운영 증적 패키지 마감

- 이유
  - 기능 구현은 끝났지만 release용 증적 보관이 아직 덜 닫혔다.
  - 계정 탈퇴 자동 worker의 실제 cron 자동 실행 1회, 커뮤니티 moderation row-level 캡처, 계정 탈퇴 최종 상태 캡처가 필요하다.
- 선행 조건
  - linked remote 접근
  - SQL/콘솔 캡처 절차

### 3. 모니터링/배포 필수 설정 최종 확인

- 이유
  - 출시 직전에는 기능보다 `관측 가능성`과 `배포 설정 누락`이 더 위험하다.
  - Sentry, Firebase 파일, Crashlytics 여부를 확인하지 않으면 장애 추적이 비어 있다.
- 선행 조건
  - Android/iOS 설정 파일 접근
  - 모니터링 대시보드 확인 가능 상태

### 4. 스토어 제출 자산 정리

- 이유
  - 지금 남은 일 중 상당수는 코드가 아니라 제출 자산이다.
  - 앱 아이콘, 스플래시, 스크린샷, 설명문, 문의처, 정책 URL을 묶어서 정리해야 제출이 가능하다.
- 선행 조건
  - PO 최종 카피 승인
  - 스토어 계정 접근 가능 상태

### 5. 출시 후보 빌드 smoke

- 이유
  - 실제 출시 직전에는 기능별 검증보다 “한 번에 끊김 없이 도는지”가 중요하다.
  - 가입, 로그인, 로그아웃, 비밀번호 재설정, 탈퇴, 커뮤니티, 기록 기본 동선을 한 번 더 묶어 볼 필요가 있다.
- 선행 조건
  - release candidate 빌드
  - 테스트 계정

## 왜 이 순서인가

- v1.0 직전에는 새 기능보다 `실기기 최종 캡처`, `운영 증적`, `모니터링`, `스토어 자산`이 더 중요하다.
- 커뮤니티와 계정 탈퇴는 기능 구현 자체보다 “문제 발생 시 설명 가능한 증적이 남는가”가 핵심이다.
- 장소/여행/산책은 문서와 emulator 기준 계약이 정리됐지만, 실제 기기 체감이 최종 출시 리스크로 남아 있다.
