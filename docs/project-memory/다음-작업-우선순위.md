1. 커뮤니티 댓글 고도화 3단계 QA 및 polish

추가 기준:

- 새 세션은 먼저 `docs/프로젝트-현황-최종기획/3차-최종-기획-진행서.md`를 읽고 시작한다.
- 단, 현재 작업 트리에 커뮤니티 미커밋 변경이 남아 있으면 먼저 그 변경이 이어지는 세션인지 확인해야 한다.
- 현재는 커뮤니티 댓글 고도화가 실제 진행 중이므로, 이 세션 체인에서는 댓글 SQL/스토어/상세 UI를 먼저 마감한다.
- 커뮤니티 작업을 계속하는 세션이 아니라면, 공식 우선순위는 계속 여행 Trust Layer와 장소 메타 쪽을 우선한다.

## 우선순위 이유

- 커뮤니티 댓글 reply 저장과 RLS recursion 이슈까지 해결됐으므로, 다음 병목은 실기기 QA와 상호작용 polish다.
- comment_like / best badge / reply 더보기 / 삭제 cascade는 실제 사용 흐름에서 한 번 더 검증해야 한다.
- 현재 `반려동물과 여행` Trust Layer 초안은 작성됐지만 아직 실제 DB 적용 전이라, 런타임 source of truth 승격이 다음 병목이다.
- 그 다음 단계가 있어야 북마크, 최근 본 장소, 운영 검수 메타를 안전하게 붙일 수 있다.
- 특히 새 펫정책 모델과 RLS 방향을 실제 Supabase 적용 순서로 정리해야 한다.
- canonical meta / source link / external signal join을 앱 코드에 안전하게 연결해야 한다.
- Google Places와 관광공사 데이터는 보조 신호 후보로만 검토하고, source of truth를 Supabase로 가져가는 방향을 유지해야 한다.
- Google Places는 상세 진입 1건 또는 상위 일부 후보에만 제한적으로 호출하는 전략을 유지해야 한다.
- TourAPI raw 문구만으로 `confirmed`를 다시 열지 말아야 한다.
- `places.primary_source` bootstrap 구조가 후속 `source_links` 확장과 충돌하지 않는지 확인해야 한다.

## 다음 작업 전 체크

- 실기기에서 현재 위치 fresh/stale 문구가 기대대로 보이는지 확인
- 펫동반 상세의 지도 미리보기/외부 지도 열기가 iOS/Android에서 모두 자연스러운지 확인
- Supabase 신규 장소 메타 스키마가 외부 API 원본을 덮어쓰지 않고 보강하는 구조인지 확인
- `pet_place_source_links` unique 전략이 Kakao / Google / TourAPI 병행에도 유지되는지 확인
- `unknown / keyword-inferred / user-reported / admin-verified / rejected` 상태가 DB 컬럼과 1:1로 맞는지 확인
- Google Places `allowsDogs` 같은 보조 속성을 실제로 붙일 경우 비용/FieldMask 범위를 통제할 수 있는지 확인
- 작성한 SQL의 RLS가 사용자 북마크/제보와 관리자 검수 요구사항을 모두 만족하는지 리뷰
- 관광공사/공공데이터를 런타임이 아닌 배치 적재로 넣을 때 정규화 기준을 먼저 정할 것
- `반려동물과 여행` 메뉴는 일상 탐색과 합치지 말고 별도 도메인으로 설계할지 확정할 것
- `반려동물과 여행` 지역명 단독 검색의 `areaBasedList2` narrow mode가 실제 기기에서 과도하게 비거나 느리지 않은지 실제 사용 흐름으로 다시 확인할 것
- `반려동물과 여행` Trust Layer DB가 `places / pet_policies / user_reports` 3축으로 자연스럽게 이어지는지 검토할 것
- `pet_travel_pet_policies`의 active source 정책과 admin review 운영 플로우를 적용 전에 다시 검토할 것
- 새 문서를 추가할 때 `docs` 역할 폴더와 `docs/sql` 정리 원칙을 계속 유지할 것
- 커뮤니티 댓글은 Android 실기기에서 reply 모드 composer가 키보드 바/IME 영역과 겹치지 않는지 반드시 확인할 것
- 커뮤니티 reply는 depth 2 이상이 실제로 저장되지 않는지 SQL + 앱 레벨에서 둘 다 검증할 것
