  - 이유: Task 7 1차는 이미 preview read-path만 좁혔고, 다음 단계도 필요 시 preview round-trip 비용만 다시 본다.

  - 이유: 공개 도메인 방어선 우선 원칙과 충돌한다.

  - 이유: 현재 5차 공개 신뢰도 원칙과 충돌한다.

## 현재 차단 및 확인 포인트

- Task 5 physical-device QA는 release blocker 유지 상태다.
- Task 4 row-level 운영 캡처와 SQL/콘솔 SOP 보강은 아직 더 필요하다.
- remote 운영 source of truth는 profile nickname path와 별도로 community path에도 생겼고, 현재 초기 seed는 `term 4건 + pattern 1건`이다.
- Task 6 create/edit/comment create UX는 emulator/manual 기준으로 닫혔고, 다음 engineering slice를 막는 항목이 아니다.
- Task 7로 list preview full eager fetch는 줄었지만, preview fetch가 아직 post별 1회라 추가 최적화 필요성은 체감 기준으로만 재판정한다.
- linked project remote에는 preview batch 전용 함수/뷰가 아직 없어, Task 7-2를 열면 app-only 미세 조정보다 서버 read contract 논의가 먼저다.
- 조회수는 `detail open + 서버 dedupe` 기본안으로 실제 구현까지 닫혔고, guest session/user 기준 dedupe와 self-view 제외가 현재 운영 계약이다.
- false positive가 날 때 운영자는 remote `community_blocked_terms`, `community_blocked_term_patterns`를 직접 조정하고 사용자 문의를 회수해야 한다.
- create/edit 이미지 upload warning 또는 image-pick 오류 노출의 수동 재현은 아직 남아 있다.
- linked temp-role push 경로가 circuit breaker에 걸려 있어, seed migration history sync는 별도 재확인이 필요하다.
- view count는 now `public.posts.view_count + community_post_view_events + record_community_post_view()` 기준의 실제 운영 계약이 있다.
- 여행 Trust Layer SQL과 linked remote runtime drift는 해소된 상태다.
- 프로필 닉네임 정책 mirror drift는 정리됐고, 남은 공백은 `supabase/migrations/*` execution history 부재다.
- 이 공백은 지금 즉시 단일 migration으로 메우지 않고, pre-Task6 auth/account baseline-history 설계로 넘긴다.
