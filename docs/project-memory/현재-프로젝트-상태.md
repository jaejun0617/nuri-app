# 현재 프로젝트 상태

## 현재 목표

NURI의 현재 목표는 기능 추가가 아니라 `App Store / Google Play v1.0 출시`다.

현재 판단 기준은 아래 3개다.

- 코드와 linked remote에 실제 계약이 반영되어 있는가
- 실기기 또는 row-level 운영 증적이 있는가
- `release-checklist.md` 기준으로 출시 게이트가 닫혔는가

## 한 줄 요약

- 핵심 기능 구현은 대부분 닫혔다.
- 인증, 탈퇴, 커뮤니티 최소 운영 방어선, 정책 문서 연결은 출시 가능한 수준까지 올라왔다.
- 별도 승인된 `건강관리 리포트 Phase 1 MVP`는 repo 구현, linked remote 반영, row-level 검증, Android 실기기 핵심 QA까지 닫혔다.
- 지금 남은 것은 새 기능 개발이 아니라 `실기기 최종 캡처`, `운영 증적 패키지`, `스토어 제출 자산`, `모니터링/배포 설정`이다.

## 운영 기준으로 닫힌 항목

### 정책 문서 연결

- 회원가입의 `이용약관`, `개인정보처리방침`, More의 `계정 삭제 안내`는 실제 public 문서로 연결됐다.
- Android 실기기 `SM_S937N`에서 문서 열림과 앱 복귀까지 PO가 직접 확인했다.
- `docs/policies/` 최신본은 Notion 공개본과 동기화된 상태로 관리한다.

### 비밀번호 재설정

- `PasswordResetRequest -> recovery link -> PasswordResetForm -> SignIn 복귀` 흐름이 구현됐다.
- Android 실기기 기준 recovery 링크 복귀, 비밀번호 변경 성공, `SignIn` 복귀까지 검증됐다.
- 기존 계정 컨텍스트로 새는 blocker는 닫혔다.

### 계정 탈퇴 7일 유예

- 탈퇴는 즉시 하드 삭제가 아니라 `요청 -> 7일 유예 -> 복구 가능 -> 자동 최종 삭제` 구조로 재편됐다.
- 유예기간 로그인 차단과 복구 모달이 앱에 반영됐다.
- 자동 파기 worker는 remote에 실제 배포됐고, `pg_cron -> Edge Function -> finalizer -> storage cleanup` 구조가 잠겼다.
- time-travel E2E 기준 due request 1건이 실제 `completed`까지 수렴했고, `profiles / pets / auth.users` 잔존 row `0`을 확인했다.

### 커뮤니티 최소 운영 방어선

- 게시글/댓글 write-path는 rate limit, dedupe, blocked-term, stable error contract를 가진다.
- 신고는 self-report 차단, duplicate 차단, reporter flag trace, unique reporter 기준 auto-hide, moderation queue/action log까지 갖는다.
- `community_image_assets`와 cleanup pending 계약이 존재하고, hidden 이미지 비노출까지 remote 재검증이 끝났다.
- `comment auto-hide`, `reporter flag trace`, `image upload/image_assets`, hidden 이미지 비노출 blocker는 닫혔다.

### 커뮤니티 인앱 정책 고지

- 목록 상단 notice, 글쓰기 helper box, 정책 링크 팝업이 반영됐다.
- Android 실기기 기준 정책 팝업과 앱 복귀를 PO가 확인했다.

### 닉네임 정책과 기본 레이아웃

- 닉네임 계약은 앱/remote 기준 `2..10`으로 정렬됐다.
- Community header, Timeline 하단 탭 유지, Android bottom gap 보정까지 실기기 검증이 끝났다.

### 장소/여행 Trust Layer 기본 계약

- 공개 라벨은 `후보 / 확인 필요 / 검수 반영` 3개로 잠겼다.
- personal state와 public trust는 분리 유지한다.
- `confirmed`는 계속 닫혀 있다.
- candidate/trust/user layer 경계는 코드와 문서에서 일치한다.

### 건강관리 Phase 1 MVP

- repo 기준으로 `HealthReportScreen`, 건강기록/체중관리/인사이트 탭, 월 단위 조회, 체중 입력 sheet, `petStore` 최신 체중 반영 경로가 구현됐다.
- linked remote 기준으로 `task7_normalize_invalid_pet_weight_snapshots`, `task7_health_report_weight_logs` migration이 적용됐고, `pet_weight_logs` table, `sync_pet_latest_weight_from_logs` function, sync/updated_at trigger가 존재한다.
- row-level 검증 기준으로 insert/update/delete/fallback 후 `pets.weight_kg`와 latest `pet_weight_logs`가 일치한다.
- Android 실기기 기준 첫 insert 직후 즉시 반영, update, delete/fallback, 홈 최신 체중 반영, 키보드 back 처리가 확인됐다.
- 건강 입력은 건강관리 전용 `기록하기`로 1차 일원화됐고, 타임라인 신규 health 작성 선택지와 생활/일정의 병원·약 신규 작성 진입점은 숨겼다.
- 인사이트 탭은 결제/잠금 없이 일반 사용자용 월간 요약과 기본 그래프를 먼저 제공한다.
- 인사이트 기록밀도는 건강 이벤트와 체중 로그를 함께 반영하며, 상단 4개 요약 위젯은 상세 리스트 바텀시트로 연결된다.
- 건강관리에서 기록 저장 후에는 저장한 기록일로 자동 포커스한다.
- 일정 추가/수정의 날짜와 시간은 `일시` 필드 하나로 묶고, 전체 화면 달력형 날짜 선택 모달 안에서 `오전/오후 + 직접 입력`으로 시간까지 함께 정한다.
- 구독 결제는 아직 도입하지 않으며, 고급 인사이트 기능 묶음은 결제 시스템과 분리해 v1.1 설계 대상으로 둔다.

## 출시 전 남은 핵심 항목

### 0. 정적 검증

- 2026-04-13 기준 `tsc`, `yarn lint`, 핵심 Jest, Android release build, Android 단독 실행 smoke는 통과했다.
- `yarn lint`에는 warning 5건이 남아 있지만 error는 0건이며, 현재 release gate 실패로 보지는 않는다.
- 최종 판정은 남은 실기기 최종 QA, 운영 증적, 스토어 자산, RC smoke 완료 여부에 따라 결정한다.

### 1. 장소/여행/산책 물리 실기기 최종 QA

- 현재 가장 큰 남은 출시 리스크다.
- 공개 라벨, `내 상태`, stale/conflict 문구, 지도 미리보기, 외부 지도 전환, 긴 설명 `더보기/접기`를 실제 기기에서 최종 캡처해야 한다.

### 2. 운영 증적 패키지 마감

- 계정 탈퇴 자동 worker의 `실제 cron 자동 tick` 1회 증적이 더 필요하다.
- 커뮤니티 moderation queue/action/image cleanup의 row-level 캡처와 release 보관본이 필요하다.
- 계정 탈퇴 최종 상태 캡처도 release pack 기준으로 정리해야 한다.

### 3. 모니터링/배포 필수 설정

- Sentry는 v1.0 Android 단독 출시에서 앱 전송 경로를 비활성화한 상태로 유지한다.
- Android `google-services.json`, Crashlytics release 설정, release signing은 확인됐고, 운영 대시보드 기준 수집 증적은 release 후보 단계에서 보강한다.
- iOS `GoogleService-Info.plist`는 v1.1 iOS 출시 항목으로 분리한다.

### 4. 스토어 제출 자산

- 앱 아이콘
- 스플래시 화면
- 스토어 스크린샷
- 스토어 설명문, 문의처, 정책 URL 최종 점검

### 5. 최종 RC smoke

- 출시 후보 빌드 기준으로 가입, 로그인, 로그아웃, 비밀번호 재설정, 탈퇴, 커뮤니티, 홈/타임라인/기록 기본 동선을 한 번 더 묶어서 봐야 한다.
