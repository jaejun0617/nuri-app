# 현재 프로젝트 상태

## 현재 목표

NURI의 현재 목표는 `핵심 task 구현 완료`다.

현재 판단 기준은 아래 3개다.

- 현재 우선 Task가 코드와 linked remote에 실제 계약으로 반영되어 있는가
- 새 도메인 전환과 기존 핵심 흐름이 서로 충돌하지 않는가
- release 문서는 유지하되, 지금 당장 닫아야 하는 lane과 후행 출시 lane이 분리되어 있는가

## 한 줄 요약

- 인증, 탈퇴, 커뮤니티 최소 운영 방어선, 정책 문서 연결, 건강관리 Phase 1 baseline은 이미 강한 기반으로 올라와 있다.
- 지금 우선순위는 release pack 마감이 아니라 남은 핵심 task를 구현 완료 상태까지 끌어올리는 것이다.
- `우리동네 동물병원`, `Guestbook -> private letters`, typography rollout, social auth, entitlement, Premium AI 선행조건이 현재 구현 lane이다.
- release checklist, evidence pack, 스토어 자산은 모든 핵심 task가 닫힌 뒤 마지막 게이트로 유지한다.

## 운영 기준으로 닫힌 항목

### 정책 문서 연결

- 회원가입의 `이용약관`, `개인정보처리방침`, More의 `계정 삭제 안내`는 실제 public 문서로 연결됐다.
- Android 실기기 `SM_S937N`에서 문서 열림과 앱 복귀까지 PO가 직접 확인했다.
- `docs/policies/` 최신본은 Notion 공개본과 동기화된 상태로 관리한다.

### 비밀번호 재설정

- `PasswordResetRequest -> recovery link -> PasswordResetForm -> SignIn 복귀` 흐름이 구현됐다.
- Android 실기기 기준 recovery 링크 복귀, 비밀번호 변경 성공, `SignIn` 복귀까지 검증됐다.
- 기존 계정 컨텍스트로 새는 blocker는 닫혔다.

### 계정 탈퇴 7일 유예

- 탈퇴는 즉시 하드 삭제가 아니라 `요청 -> 7일 유예 -> 복구 가능 -> 자동 최종 삭제` 구조로 재편됐다.
- 유예기간 로그인 차단과 복구 모달이 앱에 반영됐다.
- 자동 파기 worker는 remote에 실제 배포됐고, `pg_cron -> Edge Function -> finalizer -> storage cleanup` 구조가 잠겼다.
- time-travel E2E 기준 due request 1건이 실제 `completed`까지 수렴했고, `profiles / pets / auth.users` 잔존 row `0`을 확인했다.

### 커뮤니티 최소 운영 방어선

- 게시글/댓글 write-path는 rate limit, dedupe, blocked-term, stable error contract를 가진다.
- 신고는 self-report 차단, duplicate 차단, reporter flag trace, unique reporter 기준 auto-hide, moderation queue/action log까지 갖는다.
- `community_image_assets`와 cleanup pending 계약이 존재하고, hidden 이미지 비노출까지 remote 재검증이 끝났다.
- `comment auto-hide`, `reporter flag trace`, `image upload/image_assets`, hidden 이미지 비노출 blocker는 닫혔다.

### 커뮤니티 인앱 정책 고지

- 목록 상단 notice, 글쓰기 helper box, 정책 링크 팝업이 반영됐다.
- Android 실기기 기준 정책 팝업과 앱 복귀를 PO가 확인했다.

### 닉네임 정책과 기본 레이아웃

- 닉네임 계약은 앱/remote 기준 `2..10`으로 정렬됐다.
- Community header, Timeline 하단 탭 유지, Android bottom gap 보정까지 실기기 검증이 끝났다.

### 장소/여행 Trust Layer 기본 계약

- 공개 라벨은 `후보 / 확인 필요 / 검수 반영` 3개로 잠겼다.
- personal state와 public trust는 분리 유지한다.
- `confirmed`는 계속 닫혀 있다.
- candidate/trust/user layer 경계는 코드와 문서에서 일치한다.

### 건강관리 Phase 1 MVP

- repo 기준으로 `HealthReportScreen`, 건강기록/체중관리/인사이트 탭, 월 단위 조회, 체중 입력 sheet, `petStore` 최신 체중 반영 경로가 구현됐다.
- linked remote 기준으로 `task7_normalize_invalid_pet_weight_snapshots`, `task7_health_report_weight_logs` migration이 적용됐고, `pet_weight_logs` table, `sync_pet_latest_weight_from_logs` function, sync/updated_at trigger가 존재한다.
- row-level 검증 기준으로 insert/update/delete/fallback 후 `pets.weight_kg`와 latest `pet_weight_logs`가 일치한다.
- Android 실기기 기준 첫 insert 직후 즉시 반영, update, delete/fallback, 홈 최신 체중 반영, 키보드 back 처리가 확인됐다.
- 건강 입력은 건강관리 전용 `기록하기`로 1차 일원화됐고, 타임라인 신규 health 작성 선택지와 생활/일정의 병원·약 신규 작성 진입점은 숨겼다.
- 인사이트 탭은 결제/잠금 없이 일반 사용자용 월간 요약과 기본 그래프를 먼저 제공한다.
- 인사이트 기록밀도는 건강 이벤트와 체중 로그를 함께 반영하며, 상단 4개 요약 위젯은 상세 리스트 바텀시트로 연결된다.
- 건강관리에서 기록 저장 후에는 저장한 기록일로 자동 포커스한다.
- 일정 추가/수정의 날짜와 시간은 `일시` 필드 하나로 묶고, 전체 화면 달력형 날짜 선택 모달 안에서 `오전/오후 + 직접 입력`으로 시간까지 함께 정한다.
- 구독 결제는 아직 도입하지 않으며, 고급 인사이트 기능 묶음은 결제 시스템과 분리해 v1.1 설계 대상으로 둔다.

### 우리동네 동물병원 canonical foundation

- repo 기준으로 `animalHospital` 전용 타입, source/canonical/public/internal projection, trust/freshness 유틸, 공식 source ingest contract, runtime query service가 분리 구현됐다.
- 기존 `pet-friendly-place` 타입을 확장하지 않고, 병원 도메인 전용 public 모델만 화면에 연결했다.
- 현재 위치/최근 검색/search bar/location permission status card/map viewport 구조는 재사용하고, 병원 리스트/상세는 전용 hook과 screen route로 열었다.
- public 기본 노출은 `병원명`, `주소`, `좌표`, `거리`, `인허가/운영상태 안전 요약`, `공식/검수 전화`, `trust label`, `길찾기/전화 CTA 링크`로 제한했다.
- `운영시간`, `24시간`, `야간`, `주말`, `특수동물`, `응급`, `주차`, `장비`, `홈페이지/SNS`는 canonical/internal 구조만 만들고 public 기본 노출은 막아 두었다.
- linked remote에 `animal_hospitals`, `animal_hospital_source_records`, `animal_hospital_change_log` 최소 schema가 적용됐다.
- 공식 Localdata CSV 10,507건이 canonical/source provenance/change log에 적재됐다.
- public visible active/not hidden row는 5,427건이며 inactive row 5,080건은 public에서 제외된다.
- 공식 CSV의 EPSG:5174 좌표 9,263건은 WGS84로 확정하지 않고 `conversion-required` 상태로 남겼다.
- runtime Kakao 후보는 보수 matching으로 canonical linked candidate를 만들며, 확정 match 없는 후보명 조회 canonical은 public에 섞지 않는다.
- 동물병원 자동 외부 사진 검색은 막고, verified `thumbnailUrl`만 허용한다.
- repo 테스트 기준 animalHospital 관련 Jest 7개 suite, 16 tests와 변경 범위 ESLint는 통과했다.
- Android debug build/install/start는 성공했지만, 기기 keyguard 때문에 실제 탭 기반 smoke는 아직 미확인이다.

## 지금 당장 우선할 핵심 항목

### 1. 우리동네 동물병원 foundation

- v1.1 기준 실질 1순위 도메인이다.
- 현재 위치/지도/리스트/상세 구조 재사용 폭이 가장 크다.
- canonical foundation과 공식 source 적재는 열렸으므로, 다음은 EPSG:5174 -> WGS84 좌표 변환과 verified phone/coordinate verification contract를 먼저 닫는다.
- 운영시간/응급/특수동물/admin/reporting은 field-level verification 계약 전까지 열지 않는다.

### 2. Guestbook -> private letters 전환

- Guestbook은 실제 제품 가치가 없는 placeholder 상태다.
- 공개형 확장은 금지하고, 펫별 사적 기록 공간으로 전환해야 한다.
- 이후 Premium AI reply의 진입점이므로 IA를 먼저 고정해야 한다.

### 3. typography foundation rollout

- token과 `AppText` foundation은 있지만 실제 소비가 분산돼 있다.
- Auth, Home, Location, Modal 같은 화면군부터 순차 적용 기준을 고정해야 한다.

### 4. social auth / entitlement / Premium AI 선행조건 정리

- social login은 버튼 노출 상태일 뿐 실제 구현이 아니다.
- entitlement 없이 Premium AI나 멀티펫 구독을 열면 안 된다.
- 따라서 `Google first / Apple iOS 동시 / Kakao 후행`, `user_subscriptions + user_entitlements`, AI consent/policy를 선행조건으로 묶어 관리한다.
