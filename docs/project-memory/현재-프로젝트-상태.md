# 현재 프로젝트 상태

## 프로젝트 현재 목적

NURI는 반려동물 기록/케어 앱에서 실제 생활 행동으로 이어지는 위치 기반 기능까지 서비스 구조로 확장 중이다.

현재 위치 기반 기능의 기준은 아래 두 축이다.

- 우리동네 산책
- 펫동반 장소 탐색

## 현재 아키텍처 요약

- 앱: React Native + TypeScript
- 상태/조회: Zustand, TanStack Query
- 백엔드: Supabase Auth / Postgres / Storage
- 위치 기반 공통 레이어
  - `useCurrentLocation`
  - `useDistrict`
  - `services/location/*`
  - `services/locationDiscovery/*`

## 완료된 큰 단계

- 기록/이미지 구조 1차 안정화
- 집사 꿀팁 가이드 앱/운영 구조 1차 완료
- 산책 기능 단순화
  - 현재 GPS 기반 주변 추천 + 직접 검색
  - 현재 위치 기준 거리
  - 상세 연관 장소 더보기
  - 키보드/스크롤 안정성 보강

## 진행 중 단계

- 펫동반 장소 탐색을 실제 서비스 구조로 개방
  - 현재 위치 주변 추천
  - 직접 검색
  - 리스트/상세/지도 연결
  - 출처/검증 상태 분리

## 현재 기준 핵심 도메인 상태

- 산책
  - 공통 `locationDiscovery` 레이어 기반
  - 서비스 추천 정렬 + 거리 정렬 제공
- 펫동반 장소 탐색
  - 메뉴/화면 진입 존재
  - 현재 운영 데이터는 Kakao Local 외부 검색 결과
  - 자체 Supabase 장소 DB는 아직 없음
  - 리스트/상세/지도 미리보기/외부 지도 열기/연관 장소 더보기 구조 반영
  - `외부 후보`와 `서비스 검증 상태`를 분리한 내부 상태 모델 반영
  - `placeMeta.ts`는 향후 Supabase 메타 병합 경계로 존재하며 현재는 no-op
  - 외부 소스 역할은 `Kakao 후보 수집 / Google Places 보조 속성 / 관광공사·공공데이터 보강 / Supabase 최종 메타`로 정리됨
  - 실제 서비스급 장소 메타 SQL 문서는 `docs/펫동반-장소-탐색/데이터베이스/펫동반-장소-메타-운영-스키마-최종.sql`
  - 신뢰도 보강 API 호출 전략 문서는 `docs/펫동반-장소-탐색/펫동반-장소-탐색-신뢰도-보강-API-전략.md`

## 주요 리스크

- Kakao Local만으로는 실제 반려동물 동반 가능 여부를 100% 보장할 수 없다.
- Supabase 장소 메타 테이블이 아직 없어 운영 검수/북마크/검색 로그를 저장하지 못한다.

## 최신 운영 방향

- 위치 기반 기능은 복잡한 지역 브라우징 대신 `현재 GPS 기반 주변 추천 + 직접 검색`으로 고정한다.
- 펫동반 장소 탐색은 단기적으로 외부 API 결과를 쓰되, 중기적으로는 `외부 원본 + Supabase 메타 혼합`으로 간다.
- 펫동반 검증 상태는 `unknown / keyword-inferred / user-reported / admin-verified / rejected`로 분리하고, 카카오 결과만으로 확정 표시하지 않는다.
- Supabase 장소 메타는 `uuid PK + provider/provider_place_id unique` 구조로 설계하고, 초기에는 `text + check constraint`를 우선 검토한다.
